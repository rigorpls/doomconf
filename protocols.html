* Question Protocol

** Questions should not go in main chat!

  • Questions Should go in there own thread making them easier to ask at the end.
  • Clarifying questions should ping the host so that they can ask them as the talk is going This reduces distractions and makes finding questions easier.

** How to go

  • When beginning make a thread off #doomconf
  • Make an announcement before talks start
  • If a question is /not/ in the question thread, make a point it may not be asked / forgotten about
  • Make sure "shitposting" is not a thing, moderate and guide them to #doom-conf
  • after each talk add a large and obvious divider separating talks, making it easier for the speaker to see which questions are theres An example
```
Begining of {Speakers} Talk
```

* Recording Protocol

** Recording is important.

- We need redundency

  • multiple recorderes
  • a dedicated recording start time
  • A channel dedicated to recording logistics
  • An editor who will recive the recordings and upload it
  • - not all recordings will be needed so its up to the editor to decide what they need, it may be some people only record parts.
  • recorders and presenters can overlap

** How to go

Before conf

  • find atleast 4 recorders
  • make a thread off #doomconf

During conf

  • make sure all recorders are setup and note anyone who can't
  • give a proper start signal
  • monitor #doomconf-recording for any mishaps and pauses
  • give proper end signal

After Conf

  • share files needed with editor
